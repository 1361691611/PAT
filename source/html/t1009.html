Triple Inversions (35)

Given a list of NNN integers A1A_1A​1​​, A2A_2A​2​​, A3A_3A​3​​,...ANA_NA​N​​, there's a famous problem to count the number of inversions in it. An inversion is defined as a piar of indices i<ji < ji<j such that Ai>AjA_i > A_jA​i​​>A​j​​.

Now we have a new challenging problem. You are supposed to count the number of triple inversions in it. As you may guess, a triple inversion is defined as a triple of indices i<j<ki < j < ki<j<k such that Ai>Aj>AkA_i > A_j > A_kA​i​​>A​j​​>A​k​​. For example, in the list { 5, 1, 4, 3, 2 } there are 4 triple inversions, namely (5,4,3), (5,4,2), (5,3,2) and (4,3,2). To simplify the problem, the list AAA is given as a permutation of integers from 1 to NNN.

Each input file contains one test case. For each case, the first line gives a positive integer NNN in [3,1053, 10^53,10​5​​]. The second line contains a permutation of integers from 1 to NNN and each of the integer is separated by a single space.

Output Specification:

For each case, print in a line the number of triple inversions in the list.
